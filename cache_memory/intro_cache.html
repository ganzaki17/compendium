Motivation and Overview

Programmers ideally want unlimited fast memory, we try to create the illusion of having a large amount of fast memory by implementing memory as a hierarchy.
Multiple levels of memory having different speends and sizes are used, we keep the smaller and faster memories closer to the prcessor and the slower, larger memories further away.

Cache Memory

Cache is a small and fast buffer between the the processor and the memory.
Older unused data is replaced by newer data which is more likely to be used in the future.
Temporal Locality: If a data element is accessed, it is likely to be accessed again in the near future.
Spatial Locality: If a data element is accessed, the elements around it are likely to be accessed in the near future.

Block Placement

Block Placement answers the question where a block can be placed in the cache? There are three types of Block Placement
  1. Fully Associative
  2. Set Associative
  3. Direct Mapped
insert image here

Fully Associative

A memory block can be placed into any block in the cache.

Set Associative

A memory block can be placed into any block in a set of blocks in the cache.
(Block number) mod (Number of sets)

Direct Mapped

A memory block can be placed only into one particular block in the cache.
(Block number) mod (Number of blocks in cache)
